# Plan Generator

Прочитай PRD из файла: $ARGUMENTS

Создай план реализации и сохрани в `docs/{feature}/plan.md` — рядом с prd.md, в папке той же фичи.

## Структура плана

```markdown
# Plan: {название фичи}

**PRD:** {ссылка на файл PRD}
**Дата:** {текущая дата}

## Фазы реализации

### Фаза 1: {название}

**Цель:** что даёт эта фаза
**Затрагивает:** backend / frontend / database
**Задачи:**

- [ ] Задача 1
- [ ] Задача 2

**Когда готова:** конкретный проверяемый критерий

### Фаза 2: {название}
...
```

## Правила разбивки на фазы

- Каждая фаза даёт РАБОЧИЙ результат — можно остановиться после любой.
- Первая фаза — минимальный сквозной путь (Tracer Bullet): тонкий срез через все слои.
- Не больше 5 задач в одной фазе.
- Backend и frontend одной фичи — разные фазы.
- В каждой фазе запланированы тесты, покрывающие функционал фазы.

## Правила

- План должен покрывать ВСЕ критерии готовности из PRD.
- Не добавляй задачи, которых нет в PRD.
- Если PRD неполный — задай вопрос перед созданием плана.
