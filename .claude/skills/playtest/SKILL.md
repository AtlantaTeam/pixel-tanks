# Playtest: проверка игры в браузере

Цель: убедиться, что игровая петля живая, без ошибок и визуальных регрессий. Не «допускаем что работает» — смотрим глазами Playwright.

## Процедура

1. **Dev-сервер**: если не запущен — `npm run dev` в `web/` (порт 3050, в фоне).
2. **Загрузка**: `browser_navigate` → `http://localhost:3050/game`. Скриншот начального состояния.
3. **Консоль**: `browser_console_messages` — ноль errors. Warnings — зафиксировать в отчёте.
4. **Детерминированный выстрел**: выставить фиксированные угол/силу через UI (game-controls), выстрелить. Дождаться завершения анимации, скриншот результата.
5. **Игровая петля до конца**: серия выстрелов до победы одного танка → должен появиться game-over-dialog. Скриншот.
6. **Повторная игра**: рестарт из диалога → состояние сброшено полностью (HP, счёт, позиции).
7. **FPS/перф**: во время анимации снять `browser_evaluate` с `performance.now()`-замером кадров или прочитать счётчик FPS, если он есть. Просадка ниже 55 FPS — фиксировать как проблему.
8. **Адаптив**: `browser_resize` на мобильную ширину (390px) — канвас и контролы не разваливаются.

## Отчёт

Кратко: что проверено, скриншоты ключевых состояний, найденные проблемы (консоль, FPS, визуал) с приоритетом. Если всё зелёное — одна строка «playtest пройден» + скриншоты.
