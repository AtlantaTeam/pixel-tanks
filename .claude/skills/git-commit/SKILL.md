# Git Commit

Закоммить текущие изменения.

## Порядок действий

1. `git status --short` и `git diff --stat` — что изменилось (не читай полный diff без необходимости).
2. Сгруппируй логически связанные изменения. Несвязанные изменения — РАЗНЫЕ коммиты.
3. `git add` только нужных файлов (не `git add .` вслепую).
4. Коммит.

## Формат сообщения

```
type(scope): краткое описание в повелительном наклонении

- деталь 1 (только если нужно)

#N  ← номер Issue, если работа по Issue
```

- **type**: feat / fix / refactor / test / docs / chore / perf / ci
- **scope**: модуль или область (api, web, auth, db) — опционально
- Первая строка ≤ 72 символов, без точки в конце.
- Язык сообщения — русский (тип и scope — английские: `fix(game-engine): ...`).

## Правила

- НЕ коммить: .env и секреты, артефакты сборки, скриншоты/временные файлы.
- Если pre-commit hook упал — исправь причину и повтори. Никогда не используй `--no-verify`.
- Не добавляй `Co-Authored-By` и рекламные подписи, если в проекте не принято иное.
