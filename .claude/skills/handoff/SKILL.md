# Session Handoff

Обнови файл `.claude/state/HANDOFF.md` по текущему состоянию работы.

## Формат (жёсткий)

- Перезапиши файл ЦЕЛИКОМ — не накапливай историю.
- Максимум 50 строк.
- Структура:

```markdown
# Session Handoff

**Дата**: {сегодня, YYYY-MM-DD}

## Текущая задача

{1-3 предложения: над чем идёт работа и на каком этапе}

## Последние принятые решения

- {решение + почему, только НЕочевидные из кода/git}

## Следующие шаги

1. {конкретный шаг}

## Open questions

- {вопрос, требующий решения пользователя}
```

## Правила

- Не дублируй то, что видно из git log / status — SessionStart-хук и так это подгружает.
- Пиши только то, что следующая сессия НЕ сможет восстановить сама: намерения, отвергнутые варианты, договорённости с пользователем.
- Если разделу нечего сказать — оставь его пустым, не выдумывай.

## Коммит в main — ОБЯЗАТЕЛЬНО (строгое правило)

`HANDOFF.md` трекается и его грузит SessionStart-хук. Актуальная версия **обязана быть
в `origin/main`** — иначе она живёт незакоммиченной правкой в рабочем дереве, теряется при
`git stash`/`reset`/чистом checkout, а в main остаётся устаревшая (так и было — версия
«отставала» на дни).

- **Никогда не оставляй обновлённый `HANDOFF.md` незакоммиченным.** Обновил — сразу доведи
  до `origin/main`.
- `main` защищён → только через PR: ветка `chore/handoff-<дата>` → PR → squash-merge.
  Допустимо приложить HANDOFF к финальному PR текущей задачи, если он ещё не смёржен.
- Правки очереди раннера (`ralph.config.json`) вози **отдельной** веткой — не подмешивай
  туда HANDOFF (иначе стэшишь его и роняешь). Это разные PR.
