## Последние принятые решения

- **Claude/ralph живут в tmux-сессии `work`** (не через фоновый Bash-тул claude —
  тот умирает вместе с сессией). ПРОВЕРЕНО боевым тестом: Дима закрыл весь VS Code +
  оборвал SSH → `claude --continue` в tmux поднял разговор с полным контекстом.
  Ритуал: вернулся → `w` → `claude --continue`. Дока: `docs/ralph-prod-mode/vds-sessions.md`.
- **tmux НЕ переживает reboot VDS** (сессия в RAM, не служба). После ребута: `w`
  (создаст пустую) + `claude --continue`. Туннель (ss-local/privoxy, systemd) —
  сам поднимается. Авто-воскрешение `work` после reboot (systemd user-unit /
  tmux-continuum) — отложено в #103, НЕ сделано.
- **timeweb MCP подключён** — прямой доступ к панели (баланс, VDS, образы, удаление)
  из сессии. VDS `ralph-prod` id 8638987 статус On.
- **DI-граница runLoop/preflight**: дефолтные коллабораторы внутри читают ГЛОБАЛЬНЫЙ
  config (в проде config===cfg, ок); полностью независимый DI — вне scope, отдельного
  долга НЕ заводили (достаточно текущего).
- **Гигиена доски GitHub Projects**: воркфлоу «Item closed → Done» включён, но НЕ
  срабатывает — статусы правим руками после каждого закрытия issue. Нужна ручная
  проверка маппинга в UI проекта (через API конфиг не виден).
