## Последние принятые решения

- **Схема профилей**: `defaultProfile` + `common` + `profiles`. Мердж: объекты вглубь,
  **массивы и скаляры целиком** (частичный мердж списков двусмыслен, а догадка в
  `authorAllowlist` — дыра в защите от инъекций). Fail-closed везде, включая
  запрещённые ключи (`__proto__`/`constructor`/`prototype`) — стоп, не нейтрализация.
- **Раннер НЕ ставит обработчики сигналов — это осознанно, не забыли.** Найдено ревью
  PR #127 и подтверждено репродукцией: `process.on('SIGTERM'|…)` снимает дефолтное
  действие, а колбэк ждёт свободного event loop, которого у синхронного `runLoop` не
  бывает (spawnSync до `claudeTimeoutMs` = 2 ч). Раннер переставал умирать по Ctrl+C и
  `kill`, продолжая мерджить, а systemd видел «код 0». Уборка монитора — только на
  `'exit'`; сироту подбирает следующий запуск по `monitor.pid` (до preflight), сверяя
  `/proc/<pid>/cmdline` и на `monitor.js`, и на совпадение `--profile`.
- **Монитор запускает сам раннер** (кроме `--dry-run`), панель — `tail -f monitor.out`.
  Руками не запускать: ручной монитор не пишет pid-файл, раннер заведёт второй.
- **Ревью-конвейер, который сработал**: fable правит находки по ходу, opus смотрит PR
  свежим взглядом. Opus нашёл блокер, внесённый предыдущими проходами. Держать оба.
- **CI на PR в репо нет** — единственный workflow `deploy.yml` идёт на push в `main`.
  Гейт локальный (pre-commit + vitest); зелёного бейджа на PR не будет никогда.
  Мердж в `main` = автодеплой прода.
- **Клон был single-branch** (`remote.origin.fetch` только `main`) — из-за этого пропадала
  remote-tracking ссылка на ветку PR. Расширено до `+refs/heads/*:refs/remotes/origin/*`.
- **Токен ротирован**, новый в `ralph.env`, Fable через него работает. `update-token.sh`
  НЕ запускался (Дима вклеил руками). **Старый ключ не отозван** — только на claude.ai.
- **Fable недоступен = процесс поднят со старым токеном.** `unset` в `.bashrc` действует
  только на НОВЫЕ шеллы; tmux-сервер/vscode-server, пережившие правку, держат старый в
  памяти → пересоздать сессию. Записано в `provision/README.md`.
- **Бэкап прод-БД**: systemd timer (00:03 UTC), `docs/deploy/backup.md`. Restore проверен
  на data-level. Unit-файлы и `.bashrc` вне git — при чистом провижне вносить руками.
