## Последние принятые решения

- **Эскалация ревью — по ЗОНЕ РИСКА диффа (`review.escalateOnPaths`), не по метке
  сложности.** `complexity:expert` отвечает на «тяжело писать», а ревью усиливают там,
  где дорого ошибиться: деплой, права Payload, сам раннер. Метки как триггер оставлены
  (`escalateOn`), но в конфиге пусты. Ревью-дефолт — opus; fable дорог и на PR #127
  именно opus нашёл блокер после проходов fable.
- **`shq()` (POSIX-квотирование), а не whitelist символов** — milestone содержит
  кириллицу, скобки и «·», список разрешённых символов резал бы легальное. Но
  квотирование НЕ закрывает argument injection: значение с ведущим `-` git читает как
  опцию, поэтому `SAFE_BRANCH_RE` отдельно запрещает ведущий дефис.
  Стратегически правильнее уйти на `execFileSync` с argv (как `buildClaudeArgs`) —
  это крупный отдельный рефактор всех `sh()`.
- **Дифф в промпте ревью обрамлён делимитером и объявлен ДАННЫМИ.** Риск снижен, но не
  устранён: данные всё ещё в одном контексте с инструкциями.
- **Ревью на opus после каждой своей крупной правки себя оправдало дважды**: на #132
  нашло 2 блокера (NaN → вечный сон раннера; мёртвая зона `provision/**` при
  незакрытых `deploy/**` и `scripts/**`), на #135 — argument injection и `checksGreen`
  без валидации ветки. Свои же тесты этого не ловили: проверяли happy path и не
  сверяли, КАКАЯ команда уходит в шелл. Держать эту практику.
- **GitHub закрывает issue только по английским `closes/fixes/resolves`.** Русское
  «Закрывает #N» игнорируется молча — из-за этого #130 остался открытым. Зафиксировано
  в CLAUDE.md, ralph.md и в промпте создания PR внутри ralph.js.
- **Worktree раннера обновляется на свежий `origin/main` автоматически** при подхвате
  (`refreshRunnerWorktree`). Раньше подхватывался как есть, и кодер-сессия читала
  `ralph.md`/`ralph.js` прошлого прогона — симптом молчаливый. Грязное дерево не
  трогается: там может лежать незакоммиченная работа.
- **`main` защищена**: прямой push отклоняется, всё только через PR.
- **Ссылка на прод** (`pixeltanks.ru`) добавлена в About, topics и README.
