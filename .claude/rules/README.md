# Rules

Файлы правил подключаются в контекст ТОЛЬКО когда агент работает с файлами, совпадающими с паттерном в frontmatter (`globs`). Это экономит контекст: правило про Prisma не грузится при вёрстке.

Принцип: **чем специфичнее правило — тем точнее механизм подключения**.

- Нужно всегда → корневой CLAUDE.md
- Нужно для пакета/модуля → вложенный CLAUDE.md в его директории
- Нужно для типа файлов → сюда, в rules с `globs`
- Нужно разово → явный контекст `@файл` в промпте

Примеры в этой папке (`prisma.md`, `tests.md`) — отправная точка: отредактируй под конвенции своего проекта или удали лишнее. Итерируй по принципу «одна повторяющаяся ошибка агента — одно правило».
