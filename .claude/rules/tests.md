# Test Rules

- Тестируй поведение, а не реализацию: не мокай то, что можно вызвать по-настоящему.
- Структура: Arrange → Act → Assert, один сценарий — один тест.
- Названия описывают сценарий: `it('returns 404 when meeting does not exist')`.
- Обязательно негативные сценарии: невалидный вход, отсутствие прав, пустые данные.
- Никаких зависимостей между тестами — каждый готовит своё состояние сам.
- Запуск: `npm run test --silent` (экономия токенов на выводе).
