# Canvas / игровой движок

- Игровой цикл — `requestAnimationFrame` с delta time: физика НЕ привязана к FPS.
- Никаких аллокаций в кадре: объекты (векторы, частицы) переиспользовать (пулы), не создавать в render/update.
- Рендер и физика разделены: `model/` считает состояние, `ui/` только рисует. Физика — чистые функции, тестируемые без Canvas.
- Физика детерминирована: одинаковый вход (угол, сила, ветер) → одинаковая траектория. Random — только через инжектируемый seed.
- Canvas учитывает `devicePixelRatio` (чёткость на retina) и resize через ResizeObserver.
- Статичные слои (террейн, фон) — отдельный offscreen canvas, перерисовывать только при изменении, не каждый кадр.
- React не участвует в кадровом цикле: состояние кадра — в ref/классе движка, в Zustand — только события (выстрел, попадание, конец игры).
- `'use client'` только на компоненте-обёртке Canvas, не глубже по дереву.
