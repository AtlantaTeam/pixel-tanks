## 2. Замер FPS

### Методика

FPS замерялся на **production-сборке** — `next dev` для перфа нерепрезентативен (React
dev-overhead, HMR, отсутствие минификации). Троттлинг CPU — через CDP
`Emulation.setCPUThrottlingRate` (4× ≈ средний телефон, 6× ≈ слабый).

**Важная поправка на артефакт headless-браузера:** в фоновой вкладке Chrome троттлит
`requestAnimationFrame` до ~1 Гц, из-за чего сырые замеры давали ложные «стойки ~1006 мс».
Лечится `Emulation.setFocusEmulationEnabled({enabled:true})` — страница считается активной,
rAF идёт на частоте дисплея. Все цифры ниже сняты с этой поправкой.

Кадансовый замер упирается в частоту дисплея тестового стенда (~170 Гц), поэтому истинный
запас показывает **синхронная стоимость работы игрового колбэка** (замер длительности каждого
вызова `animate`), не зависящая от частоты обновления.

### Результаты — стоимость работы на кадр (мс), выстрел + все эффекты

| Профиль              | median | p95 | max     | кадров >16.6 мс | кадров >33 мс |
| -------------------- | ------ | --- | ------- | --------------- | ------------- |
| Desktop (1×)         | 0.1    | 0.4 | **0.5** | 0               | 0             |
| Средний телефон (4×) | 0.3    | 1.8 | **4.6** | 0               | 0             |
| Слабый телефон (6×)  | 0.3    | 2.4 | **4.5** | 0               | 0             |

Бюджет: 16.6 мс на кадр = 60 FPS, 33 мс = 30 FPS. Даже под 6× троттлингом худший кадр
(4.6 мс) укладывается в бюджет 60 FPS с 3.5× запасом. Число «рабочих» кадров растёт под
троттлингом (161 → 311) — это slow-mo растягивает окно эффектов, но каждый кадр остаётся
дешёвым. Причина запаса: инкрементальный redraw (`explosionAreaRedraw`/`tankAreaRedraw` вместо
`fullRedraw` каждый кадр), пул частиц без аллокаций в кадре, затухающий трейл с самоочисткой.

**Кадансовый замер (для полноты):** desktop/4×/6× — ~170 FPS (упор в дисплей), 0 кадров >16.6 мс
при 1×/4×, 0 кадров >33 мс при 6×.

## 4. Прочие наблюдения playtest

- Консоль: игровых ошибок нет. Присутствуют только `favicon.ico` 404 (безобидно) и предупреждение
  `AudioContext was not allowed to start` — ожидаемо (звук стартует после жеста пользователя,
  autoplay-политика; так и задумано в Фазе 5).
- Мобильные вьюпорты: **375×667** (портрет) и **667×375** (ландшафт) — горизонтального скролла нет,
  HUD целиком, тач-цели крупные.
- Диалог конца боя и «Новая игра» (reset + `window.location.reload()` → полный сброс) —
  верифицированы по коду и стору; автопрогон полного боя ботом-vs-игрок нестабилен по таймингу
  смены ходов (не баг движка).
- ⚠️ Инфра-заметка (вне scope #25): `next start` выдаёт предупреждение о несовместимости с
  `output: standalone` в `next.config`. На перф игры не влияет; для прод-деплоя стоит запускать
  `node .next/standalone/server.js`. Кандидат в отдельный chore.
