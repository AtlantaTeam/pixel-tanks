# Plan: game-next — некст-ген апгрейд игры

**PRD:** [prd.md](./prd.md)
**Дата:** 2026-07-18

## Фазы реализации

### Фаза 1: Детерминированная физика + seed (Tracer Bullet)

**Цель:** фундамент под daily challenge, реплеи и тесты — одинаковый seed всегда даёт одинаковый бой
**Затрагивает:** frontend (game-engine)
**Задачи:**

- [ ] Инжектируемый seeded-random в game-engine (террейн, ветер) вместо `Math.random`
- [ ] Seed принимается через параметр URL `/game?seed=...`, без параметра — случайный
- [ ] Детерминированные unit-тесты физики: фиксированный вход (угол, сила, ветер) → фиксированная траектория
- [ ] Unit-тесты генерации террейна: один seed → идентичный массив высот

**Когда готова:** два открытия `/game?seed=42` дают идентичный террейн и ветер; `npm run test` зелёный (первые тесты в проекте)

### Фаза 2: Мобильный layout

**Цель:** игра корректно выглядит и не разваливается на телефоне (пока без тач-управления)
**Затрагивает:** frontend
**Задачи:**

- [ ] Вьюпорт: `h-dvh`, safe-area insets, `touch-action: none` на Canvas, запрет горизонтального скролла
- [ ] Адаптивный HUD: портрет и ландшафт 375px, все тач-цели ≥ 44px
- [ ] Canvas: `devicePixelRatio` + resize через ResizeObserver
- [ ] Playwright-проверка вьюпортов 375×667 и 667×375: нет h-скролла, HUD виден целиком

**Когда готова:** скрины 375px (портрет + ландшафт) без горизонтального скролла, HUD целиком, тач-цели ≥ 44px

### Фаза 3: Тач-управление «оттяни и отпусти»

**Цель:** полноценная игра пальцем на телефоне — приоритет PRD
**Затрагивает:** frontend (game-engine)
**Задачи:**

- [ ] Жест: тянешь от танка — направление задаёт угол, длина — мощность; отпустил — выстрел
- [ ] Превью-линия прицела при оттяжке
- [ ] Защита от конфликтов жестов (скролл/зум не срабатывают во время прицеливания)
- [ ] Playwright e2e с touch-эмуляцией: полный бой с ботом от старта до победы на 375px

**Когда готова:** e2e-тест «полный бой жестом на 375px» зелёный в портрете и ландшафте

### Фаза 4: Клавиатурная схема

**Цель:** полный бой без мыши — как в старой игре
**Затрагивает:** frontend (game-engine)
**Задачи:**

- [ ] Стрелки — точная настройка угла/мощности; Ctrl+стрелки — смена оружия / перемещение танка; Enter/Space — выстрел
- [ ] Экранная подсказка раскладки (уже заявлена на главной — сверить с фактической схемой)
- [ ] Playwright e2e: полный бой с ботом только клавиатурой

**Когда готова:** e2e-тест «бой только клавиатурой» зелёный; три схемы (тач/клавиатура/мышь) не конфликтуют

### Фаза 5: Оригинальный звук

**Цель:** вернуть мелодии Андрея и звуки боя
**Затрагивает:** frontend
**Задачи:**

- [ ] Скопировать аудио из `../pocket-tanks/static/audio/` в `public/audio/`
- [ ] Аудио-модуль на WebAudio: музыка меню (`game-menu.mp3`), музыка боя (`gameplay.mp3`), эффекты (`fire.wav`, `explosion-hit.wav`, `explosion-miss.wav`)
- [ ] Запуск звука только после первого жеста пользователя (autoplay policy)
- [ ] Mute-переключатель в HUD, состояние в localStorage (переживает перезагрузку)
- [ ] Unit-тесты аудио-модуля (моки WebAudio: правильный трек на правильное событие)

**Когда готова:** в меню и бою играет оригинальная музыка, выстрел/взрывы озвучены, mute работает и сохраняется

### Фаза 6: Juice-пакет

**Цель:** игра ощущается «сочной» — визуальный фидбек каждого действия
**Затрагивает:** frontend (game-engine)
**Задачи:**

- [ ] Пул частиц: взрыв земли, вспышка урона на танке (без аллокаций в кадре)
- [ ] Screen shake при взрыве, slow-mo при попадании в танк
- [ ] Трейл снаряда; анимация обновления счёта/ходов в HUD
- [ ] `prefers-reduced-motion`: shake/slow-mo отключаются
- [ ] Прогон скиллов `game-feel` + `playtest`: FPS ≥ 60 desktop, ≥ 30 на среднем телефоне (CPU throttle)

**Когда готова:** чек-лист `game-feel` пройден, FPS-замеры в норме, reduced-motion уважается

### Фаза 7: Бот-реплики (Терминатор 2.0)

**Цель:** вернуть характер бота из старой игры
**Затрагивает:** frontend (game-engine)
**Задачи:**

- [ ] Перенести пул реплик из `../pocket-tanks/src/modules/bot-messages/` (≥ 15 реплик: злой/довольный/сарказм)
- [ ] Chat-bubble в пиксельном стиле поверх танка бота (DOM-слой над Canvas)
- [ ] Привязка к событиям: попадание бота, попадание игрока, самострел/нелепый промах
- [ ] Unit-тесты выбора реплики по типу события

**Когда готова:** каждое событие попадания/промаха вызывает bubble с репликой нужной категории

### Фаза 8: Daily challenge

**Цель:** «Бой дня» — общий seed на всех, повод возвращаться
**Затрагивает:** frontend + backend (Payload)
**Задачи:**

- [ ] Дневной seed: детерминированная функция от даты (UTC), режим «Бой дня» с главной
- [ ] Запись результата боя дня в Payload `scores` (Local API, с пометкой daily-seed)
- [ ] Экран результата: счёт + приглашение поделиться
- [ ] Тесты: функция seed-от-даты, запись результата (валидные/невалидные данные)

**Когда готова:** два браузера в одни сутки получают идентичный террейн/ветер; результат сохраняется в БД

### Фаза 9: Реплеи-ссылки

**Цель:** шаринг боя ссылкой — смотрится без регистрации
**Затрагивает:** frontend
**Задачи:**

- [ ] Запись боя: seed + последовательность ходов → компактная строка (URL-safe)
- [ ] Страница `/replay/[code]`: покадровое воспроизведение через движок
- [ ] Кнопка «Поделиться» после боя (Web Share API + fallback копирования)
- [ ] Тест детерминизма: сериализованный бой → воспроизведение → идентичный итоговый счёт

**Когда готова:** ссылка реплея на чистом браузере воспроизводит бой с тем же итоговым счётом
